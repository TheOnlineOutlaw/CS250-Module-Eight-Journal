# How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?
Interpreting user needs will generally start with an analysis of the system.  This can be done by doing a number of things such as asking the team for feedback and asking previous users for any errors or changes that they have seen within the code. All of the inputs and outputs should also be identified as well. User stories can help with this by gathering information. This information is used to find what the users needs and wants are. Looking at the user stories will help us understand what the users are looking for in the product.

# How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?
I approach developing programs in a couple different ways. The Waterfall Method is one strategy that works well with a linear project that is not expecting changes once started. The other method is the Agile Method. The Agile offers a more flexible strategy and it is a more unique way to work with testing, planning and client integration. This allows for changes to be easily added if they come up during the project.

# What does it mean to be a good team member in software development?
To me being a good team member means to make sure you are understanding your part of the project and making sure you are keeping up with your part and not lagging behind, which will make the others on the team behind., I also think being a good team member means to listen to your other co-workers and to help anyone who may need it.
